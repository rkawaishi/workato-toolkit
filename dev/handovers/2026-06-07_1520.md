## 決定事項
- 旧 `workato-dev-kit` は**開発停止(凍結)**。今後は `workato-toolkit` のみで開発。
- **マルチエディタ4種維持**。**共有ツリー型**(root 自体が plugin、`skills/docs/bin/agents/*.md` を1式共有、manifest が `source:"./"` を指す)。**`dist/` 複製は作らない**。
- 派生のみエディタ別: hooks(3種)・Cursor `.mdc`・Codex agent `.toml`・context・`.mcp.json`。
- **docs 参照 = docs-overlay MCP に統一**(skill 本文は plugin-root 変数も素の相対パスも使わず MCP tool を呼ぶ)。kit docs + project `org/docs/` を org-wins merge。これで #1(同梱 docs 参照)・org overlay・学習書き戻し・project-dir 解決 を一挙に解決。
- 顧客=**Workato 導入の中〜大企業(組織利用)**。価値の柱=**ナレッジ共有(スコープはチーム内=1 workspace repo)**。配布=**public OSS**、ローンチ=**MVP 優先**(MVP=CC+Cursor+Codex 先行、Gemini は実機検証後)。エンタープライズ特有要件(厳密な版固定/監査/SSO)は今回非重視。
- MCP は **Python(FastMCP)**。新 repo は P2 から **feature ブランチ + PR**(P1 のみ新規 repo 初期化のため main 直コミット)。

## 捨てた選択肢と理由
- 「submodule + setup.sh 継続」「submodule と plugin の併設」「2 repo(canonical 別 + 生成 publish)」「エディタ別 `dist/` 生成」「CC 一本化」→ いずれも不採用(PR/FAQ §2.2 に比較表)。理由は導入煩雑/二重メンテ/複製不要(共有ツリーで足りる)/マルチエディタ維持要件。
- docs 参照を「per-editor stamp(skill の docs 行だけ生成)」案 → MCP の方が4エディタ共通・overlay 一元化・学習も載るため不採用。
- repo 名は造語/初期化語(wdk/wkit/wkat 等)→「分かりづらい」で却下、`workato-toolkit` に決定。

## ハマりどころ
- Claude Code 等の plugin は skill 本文の**素の相対パスを「ユーザー cwd」基準**で解決し、同梱 docs を指さない。plugin-root 変数はエディタ別 → 単一 SKILL.md 共有と衝突。→ **MCP 経由参照**で回避(P2 の核心)。
- Bash の cwd ドリフト注意。git 操作前に `git rev-parse --show-toplevel` で repo を確認した(誤って workato-dev-kit を触らないため)。
- Codex marketplace の `source` は `.agents/plugins/marketplace.json` で `{source:"local", path:"./"}`、Cursor marketplace は `"."`、CC/Codex skills は `"./"` 系 — archcore 実値で確定。
- MCP の依存配布は `uv run --with fastmcp`(zero-install)。`uv 0.11.2` / `python 3.14` で動作確認済。`uv` 不在環境では起動方式の切替が必要。
