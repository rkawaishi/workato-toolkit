# Genie and MCP build guide

Steps for building and deploying Workato's AI agents (Genie) and MCP servers. The same skills and the same steps work in both Claude Code and Cursor.

## Overview

### Genie (Agent Studio)

Genie is Workato's AI agent platform. It lets users run business workflows in natural language from Slack or Teams.

**Four pillars:**
1. **AI model and job description** — defines the agent's role and behavior
2. **Chat interface** — used from Slack, Teams, or Workato GO
3. **Knowledge base** — references documents such as PDF, CSV, and Markdown
4. **Skills** — calls Workato recipes as "tools"

### MCP server

A server compliant with the Model Context Protocol (MCP). It lets external AI models (such as Claude Desktop) call Workato skills as tools.

**Three types:**
- **Workato-hosted** — an MCP server provided by Workato
- **Local** — an MCP server that runs in a local environment
- **Developer API** — an MCP server accessed via API

## Components

```
<project>/Agents/
├── <genie>.agentic_genie.json     # Genie definition
├── <skill>.agentic_skill.json     # Skill definitions (one or more)
├── <mcp>.mcp_server.json          # MCP server definition (optional)
└── ...

<project>/Recipes/
├── <skill-recipe>.recipe.json     # Recipe invoked by the skill
└── ...
```

### Genie definition (`.agentic_genie.json`)

- **instructions**: instructions for the agent (role, constraints, response style)
- **skills**: references to the skills bound to the Genie
- **ai_model**: AI model configuration to use

### Skill definition (`.agentic_skill.json`)

- **parameters**: parameter definitions that receive user input
- **recipe_reference**: reference to the recipe to execute
- **trigger_type**: the skill's trigger type

### MCP server definition (`.mcp_server.json`)

- **tools[]**: list of tools to expose (description, VUA requirement)
- **references**: mapping of references to skill definitions

For JSON formatting rules, see `.claude/rules/workato-agentic-format.md` (Cursor: `.cursor/rules/workato-agentic-format.mdc`).

## Build steps

### 1. Design

Use `/spec` then `/plan` to consolidate the overall agent design in `plan.md`:

- **Persona**: who the agent is for (sales team, IT help desk, etc.)
- **Skill list**: what operations it can perform
- **Connections**: which systems it connects to
- **MCP exposure**: whether to make it usable from external AI as well

### 2. Asset generation

```
/create-genie
```

Interactively decides and generates the following:

1. The Genie's purpose and target users
2. Required skills (units of operation)
3. Parameters for each skill
4. AI provider (default: Workato-provided model)
5. Whether to expose as an MCP server
6. The Genie's instructions (auto-generated)

Skill recipes are generated by delegating to `/create-recipe`.

### 3. Deploy

```
/push-project
```

Post-push verification:

```
Step 1: Verify the Genie in Agent Studio
  -> Are the skills recognized?
  -> Are the instructions set correctly?

Step 2: Test
  -> Invoke skills from the chat interface
  -> Verify that parameters are passed correctly

Step 3: (For MCP) Verify tool exposure
  -> Verify the MCP server endpoint
  -> Run a connection test from an external AI client
```

## Design tips

### Designing Genie instructions

Instructions are the single most important factor determining agent quality:

- **Clarify the role**: "You are an IT help desk assistant."
- **Set constraints**: "Do not access HR information directly; always go through the approval flow."
- **Response style**: "Answer concisely and avoid technical jargon."
- **Conditions for skill use**: "Run a skill only when the user explicitly requests it."

### Designing skill granularity

- **One skill = one operation** is the basic rule ("Create a Jira ticket", "Notify Slack")
- For compound operations, define the combination in the Genie's instructions
- Keep parameters to the minimum necessary. Do not make a value a parameter if the AI can infer it.

### Deciding whether to expose via MCP

| Expose via MCP | Do not expose via MCP |
|---|---|
| You want external AI (Claude Desktop, etc.) to use it | The Slack/Teams Genie chat is enough |
| You want multiple AI clients to share it | The audience is only in-house Workato users |
| You want to publish it externally as an API | External exposure is not allowed for security reasons |

### Security

- **Verified User Access (VUA)**: can require user authentication when running a skill
- **RBAC**: control access to the Genie and skills by role
- **Audit logs**: every skill execution is recorded
- **OAuth authentication**: connections to the MCP server are protected by OAuth

## Calling a Genie from a recipe

You can also assign tasks to a Genie from within a recipe:

- Connector: `workato_genie`
- Action: `assign_task_to_genie`
- Use case: delegating steps that require judgment to the AI (for example, classifying emails or summarizing data)

## Known limitations

- `/push-project --delete` cannot delete `agentic_skill` or `mcp_server`. Manual deletion from the UI is required.
- When you bind a skill to an MCP server, the skill name may be auto-renamed.
- Knowledge base file uploads can only be done from the UI.
